What This Project Is About

Project 1 is teaching your team how distributed messaging works using the publish-subscribe model. In the final version, your sender (producer) will publish messages to Google Pub/Sub, and your consumer will read them from a subscription and save them to a database.

The key concept your instructor wants to see is at-least-once delivery: a message may be delivered more than once, so the consumer must detect duplicates before inserting rows into the database.

Required Components (Final Project)

  • Google Cloud Project + Pub/Sub Topic + Subscription
  • Producer component + sender UI
  • Consumer component with duplicate detection before DB insert
  • Database storage
  • Separate viewer UI with sorting/filtering and duplicate highlighting

What This Milestone 3 Shell Demonstrates

  • Separate Login, Sender, and Receiver/Sorter pages
  • Standalone producer script (local outbox simulation)
  • Standalone consumer script (reads outbox, writes SQLite)
  • Duplicate detection and highlighted duplicate rows
  • Sorting and filtering in the viewer
